Null and Undefined

למשתנים אין תמיד ערך. כדי לייצג חוסר השמה משתמשים בnull ו undefined.

Undefined

כאשר אנו מצהירים על משתנה מבלי לבצע השמה. הערך והטיפוס של המשתנה הם undefined. 

                    

    var noValue;
    console.log(noValue); // undefined
    console.log(typeof noValue); // undefined

גם במקרים אחרים מתפקד undefined, לדוגמא כאשר אנו מנסים לגשת לאינדקס לא קיים במחרוזת: 

                    

    var name = "Peter";
    console.log(name[0]); // Console will show "P"
    console.log(name[4]); // Console will show "r"
    console.log(name[10]); // Console will show undefined

כאשר אנחנו מדברים על מערכים ואובייקטים אנו נראה ש:

  • כמו במחרוזות, למערכים יש אורך, ולכל אלמנט במערך יש אינדקס בעזרתו אפשר לגשת אליהם. כאשר ננסה לגשת לאינדקס מחוץ למערך, נקבל את הערך undefined.

  • לאובייקטים יש שדות (תכונות). לאובייקט מכונית כנראה יהיה שדה "יצרן" או "סוג" אבל כנראה שלא יהיו לו שדה "אימייל". אם ננסה לגשת לשדה לא קיים של אובייקט, נקבל את הערך undefined.

    • Null

    null הוא לא טיפוס, אלא אובייקט. ומסיבות כאלו ואחרות אנשים ממליצים שלא להשתמש בו בJavascript.

    השימוש בnull הוא בעיקר בשביל לתת ערך שמשמעותו "כלום" למשתנה עם השמה קודמת: 

                        

    var temperature = 35;
    console.log(temperature);

    temperature = null;
    console.log(temperature); // The console will show null
    console.log(typeof temperature); // The console will show object - which could be misleading

    עם זאת, את אותה פעולה אפשר לעשות עם שימוש בundefined: 

                        

    var temperature = 35;
    console.log(temperature);

    temperature = undefined;
    console.log(temperature); // The console will show undefined

    למעשה, Javascript מחשיבה את null ואת undefined כשווי ערך: 

                        

    console.log(undefined == null);  // The console will show true

    חלק מהאנשים ככל הנראה משתמשים בnull בשביל להפריד בין המקרים בהם יש למשתנה ערך קודם שאותו רוצים "לאפס".

    בדוגמאות שלנו לא נשתמש בnull אלא בundefined בלבד.
