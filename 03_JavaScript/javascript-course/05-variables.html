תוכנית ממר"ם

משתנים

תוכניות מחשב קיימות בשביל לעשות מניפולציות על המידע. מה שהן עושות בעקרון הוא לקבל קלט, לעבד אותו ולהראות פלט כלשהו. עד כה לא ראינו את האתר שלנו כתוכנית מחשב כיוון שהיו לנו רק דפים סטטים. אך מעתה ואילך, אם נרצה שהאתר שלנו יהיה דינאמי ואינטרקטיבי נצטרך לחשוב עליו כתוכנת מחשב שרצה על האינטרנט.

הסתכלו בדוגמא הבאה אשר מקיימת אינטרקציה עם המשתמש.



בדוגמא הזאת אנחנו עובדים עם מידע. מקבלים קלט מהמשתמש, מעבדים את הודעת הgreeting ומראים אותה בתוך אלמנט HTML. קיימת בנוסף גם לוגיקה, כיוון שאם השדה ישאר ריק, נחליף את הקלט בguest user.

כדי לעבוד עם מידע בתוכנת מחשב, אנחנו צריכים משתנים.

משתנים נועדו לשמור מידע שבו נשתמש לאחר מכן. לכן ניתן למשתנים שמות כדי שנוכל להתייחס אליהם. השם של משתנה נקרא גם מזהה.

יצרנו משתנה באמצעות המילה "var" ולאחריה השם שלו. לאחר מכן ביצענו השמה למשתנה באמצעות הסימן =. 

                    
    var user_name = "guest user";

לאחר השמת המחרוזת "guest user" למשתנה, נוכל לגשת אליו בכל זמן שנרצה. כדי לבדוק זאת, ניצור משתנה כמו הנראה למעלה, ולאחר מכן נריץ את הstatement: 

                    
    console.log(user_name);

בשביל מה לשלוח את הערך הנ"ל לconsole? זאת דרך טובה לבדוק את הקוד שלנו. כעת כשאנחנו יודעים שהמשתנה שלנו נוצר והערך הושם בהצלחה למשתנה, נוכל להשתמש בו כדי ליצור את הודעת הgreeting ולשים את ערכו באלמנט הHTML. 

                    
    document.getElementById("user_greeting_message").innerHTML = "Hello " + user_name + "!";

כדי ליצור את ההודעה הנראית למעלה, אנחנו מבצעים שרשור מחרוזות. בהמשך נראה מבני נתונים בJavascript.

לפני שנמשיך, גם בJavascript יש לשמות המשתנים חוקים.

שמות משתנים

שמות משתנים יכולים להכיל:

  • אותיות
  • מספרים
  • קו תחתי (_)
  • תו דולר ($)

דגש חשוב: משתנים יכולים להגיל מספרים אבל הם לא יכולים להתחיל במספר. רווחים גם הם לא חוקיים.

כמה דוגמאות לשמות משתנים:

                        
    a             // ok
    
    name          // ok
    
    $name         // ok
    
    students_name // ok
    
    student_1     // ok
    
    1_b           // invalid because it starts with a number
    
    my-name       // invalid because it has a dash (only underscores are allowed)
    
    my name       // invalid because it contains a white space
    
    new           // invalid because 'new' is a Javascript reserved word, like will be explained below.

הימנעו משימוש באותיות בעברית. למרות שהקוד ירוץ, עדיף לכתוב קוד שיובן לכל מי שדובר אנגלית.

מילים שמורות

כמו בשפות אחרות, ישנן מילים שמורות שנועדו לפקודות ופונקציות ולכן לא יוכלו לשמש כשמות למשתנים. דוגמא לכמה מילים שמורות:

  • function
  • if
  • else
  • var
  • new
  • return
  • break
  • default
  • in
רשימה של כל המילים השמורות אפשר לראות כאן

רגישות לאותיות גדולות/ קטנות

כמו שהוסבר קודם, Javascript רגישה לאותיות גדולות/ קטנות, אז לא לשכוח זאת כשאשר יוצרים משתנים. אם ניצור משתנה בשם "username" ואז ננסה לקרוא לו עם "Username" נקבל הודעת שגיאה.

כדאי לתת לשמות המשתנים שמות משמעותים המעידים על התפקיד שלהם

קוד טוב צריך להיות מובן בקלות, לא רק למי שכתב אותו אלא גם עבור אנשים אחרים, אז קחו בחשבון כשאר אתם בוחרים שמות למשתנים. אם נכתוב תוכנית המקבלת כקלט שם פרטי ושם משפחה ממשתמש, אלו מהשמות הבאים כדאי שנשתמש: 

                    
    //Option # 1:
    var first_name = "John";
    var last_name = "Doe";

    //Option # 2:
    var name1 = "John";
    var name2 = "Doe";

    //Option # 3:
    var a = "John";
    var b = "Doe";

קל לראות שהאפשרות הראשונה היא הטובה ביותר. האפשרות השניה סבירה אבל לא לגמרי טובה. האפשרות השלישית גרועה מאוד.

מוסכמה לשמות

לשפות תכנות יש בדרך כלל מוסכמה לכתיבת שמות כדי שהקוד יהיה קריא יותר. Javascript לרוב תבחר במוסכמה של lower camelCase בשביל שמות משתנים. הכוונה היא שהאות הראשונה היא קטנה והאות הראשונה של המילה הבאה היא גדולה. 

                    
    // camelCase examples:
    
    firstName
    
    lastName
    
    exchangeRate
    
    maxContainerWidth
    
    screenSize

בנוסף נפוץ שימוש בכתיבת השמות באותיות קטנות כאשר ההפרדה בין מילים היא עם קו תחתי, כמו last_name או max_container_width. תבחרו איזה סגנון שמרגיש לכם יותר נכון, העיקר שתישארו עקביים. בנוסף, נסו לשמור על שמות קצרים ככל הניתן, כלומר השם firstName טוב יותר מאשר השם firstNameOfTheUser.
